บทที่ 9 : PHP Interfaces

อินเทอร์เฟซ (PHP Interfaces)

การสร้างข้อตกลงและมาตรฐานในการพัฒนาด้วยคำสั่ง interface และ implements พร้อมข้อดีที่รองรับการทำ Multiple Inheritance

Interface คืออะไร?

Interface คือ "สัญญาหรือข้อตกลง (Contract)" ที่กำหนดว่าคลาสใดก็ตามที่นำอินเทอร์เฟซนี้ไปใช้ จะต้องมีเมธอดอะไรบ้าง โดยเราใช้คำสั่ง interface ในการสร้าง และใช้คำสั่ง implements ในคลาสที่จะนำไปใช้

ความสุดยอดของ Interface คือ คลาส 1 คลาสสามารถ implements ได้หลายอินเทอร์เฟซพร้อมกัน (คั่นด้วยเครื่องหมายจุลภาค ,) ซึ่งช่วยแก้ข้อจำกัดของ PHP ที่ไม่สามารถ extends หลายคลาสได้

ข้อแตกต่าง: Interface vs Abstract Class

  • Interface: ห้ามมี Properties, ทุกเมธอดต้องเป็น public และห้ามมีเนื้อหาในเมธอดเด็ดขาด (มีได้แค่ชื่อเมธอด)
  • Abstract Class: สามารถมี Properties ได้, เมธอดเป็น public หรือ protected ก็ได้ และสามารถมีเมธอดธรรมดาที่มีเนื้อหาการทำงานผสมอยู่ได้

ตัวอย่างการใช้ interface และ implements

PHP 8+
interfaces.php 
<?php
// 1. สร้าง Interface ชื่อ Animal (กำหนดว่าสัตว์ทุกตัวต้องร้องได้)
interface Animal {
  public function makeSound();
}

// 2. คลาส Cat นำอินเทอร์เฟซ Animal ไปใช้ด้วยคำสั่ง implements
class Cat implements Animal {
  public function makeSound() {
    echo "🐱 แมวร้อง: เหมียว เหมียว! <br>";
  }
}

class Dog implements Animal {
  public function makeSound() {
    echo "🐶 สุนัขร้อง: โฮ่ง โฮ่ง! <br>";
  }
}

// สร้างออบเจกต์และวนลูปเรียกใช้งานตามมาตรฐานเดียวกัน (Polymorphism)
$cat = new Cat();
$dog = new Dog();
$animals = [$cat, $dog];

foreach ($animals as $animal) {
  $animal->makeSound();
}
?>
