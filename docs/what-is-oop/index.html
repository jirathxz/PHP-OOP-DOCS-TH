บทที่ 1 : PHP What is OOP

OOP ในภาษา PHP คืออะไร?

ทำความเข้าใจความแตกต่างระหว่างการเขียนโปรแกรมแบบขั้นตอน (Procedural) และเชิงวัตถุ (OOP) พร้อมข้อดีที่ทำให้โปรแกรมเมอร์ทั่วโลกเลือกใช้

OOP คืออะไร?

OOP (Object-Oriented Programming) คือแนวคิดการเขียนโปรแกรมที่มองทุกอย่างให้เป็น "วัตถุ (Object)" ซึ่งประกอบไปด้วย 2 ส่วนหลักคือ:

  • Properties (คุณสมบัติ/ตัวแปร): ข้อมูลที่เก็บไว้ในออบเจกต์ เช่น ชื่อ, สี, ราคา, น้ำหนัก
  • Methods (พฤติกรรม/ฟังก์ชัน): สิ่งที่ออบเจกต์นั้นสามารถทำได้ เช่น คำนวณ, ขับเคลื่อน, บันทึกข้อมูล

Procedural vs Object-Oriented

📝 Procedural Programming

เขียนโค้ดเป็นลำดับขั้นตอน คำสั่ง และฟังก์ชันเรียงต่อกันไป เหมาะกับโปรแกรมขนาดเล็ก แต่อาจจัดการและแก้ไขยากเมื่อระบบใหญ่ขึ้น

📦 Object-Oriented Programming

จัดกลุ่มข้อมูลและฟังก์ชันเข้าไว้ด้วยกันเป็น Object ทำให้โครงสร้างโค้ดชัดเจน ปลอดภัย และนำกลับมาใช้ซ้ำ (DRY) ได้อย่างมีประสิทธิภาพ

ข้อดีของ PHP OOP (อิงจาก W3Schools)

⚡ รวดเร็วและง่ายต่อการใช้งาน OOP ช่วยให้การประมวลผลและการจัดโครงสร้างโค้ดเป็นระบบมากขึ้น ทำให้การพัฒนาโปรแกรมรวดเร็วขึ้น
🏗️ โครงสร้างชัดเจน (Clear Structure) กำหนดขอบเขตและหน้าที่ของแต่ละคลาสอย่างชัดเจน ไม่ปะปนกัน (Modularity)
♻️ หลักการ DRY (Don't Repeat Yourself) ช่วยให้คุณไม่ต้องเขียนโค้ดเดิมซ้ำๆ สามารถสืบทอด (Inheritance) และนำโค้ดกลับมาใช้ใหม่ได้ทันที
🔧 บำรุงรักษาและขยายระบบง่าย เมื่อต้องการแก้บั๊กหรือเพิ่มฟีเจอร์ สามารถทำได้เฉพาะจุดโดยไม่ส่งผลกระทบให้โปรแกรมส่วนอื่นพัง

ตัวอย่างแนวคิด Class และ Object ใน PHP

PHP 8+
concept.php 
<?php
// 1. สร้าง Class (แม่พิมพ์) ชื่อ Fruit
class Fruit {
  // Properties (คุณสมบัติ)
  public $name;
  public $color;

  // Methods (พฤติกรรม/การทำงาน)
  function set_name($name) {
    this->name = $name;
  }
  function get_name() {
    return this->name;
  }
}

// 2. สร้าง Object (ผลงานจริง) จาก Class Fruit
$apple = new Fruit();
$banana = new Fruit();

// กำหนดค่าและเรียกใช้งาน
$apple->set_name('Apple');
$banana->set_name('Banana');

echo $apple->get_name();  // ผลลัพธ์: Apple
echo "<br>";
echo $banana->get_name(); // ผลลัพธ์: Banana
?>
