CORE CONCEPTS

4 เสาหลักของแนวคิดเชิงวัตถุ (The 4 Pillars of OOP)

หัวใจสำคัญ 4 ประการที่ทำให้โครงสร้างโปรแกรมมีประสิทธิภาพ เป็นระเบียบ และขยายต่อได้ง่าย

1. Encapsulation

การห่อหุ้มข้อมูล

รวบรวมข้อมูลและพฤติกรรมไว้ในออบเจกต์ พร้อมใช้ Access Modifiers (public, private, protected) ป้องกันการเข้าถึงและแก้ไขข้อมูลโดยไม่ได้รับอนุญาต

2. Inheritance

การสืบทอดคุณสมบัติ

ช่วยลดการเขียนโค้ดซ้ำ (DRY) โดยให้คลาสลูกสามารถรับเอาคุณสมบัติและเมธอดจากคลาสแม่ผ่านคำสั่ง extends เพื่อขยายการทำงานต่อได้ทันที

3. Polymorphism

พ้องรูป / การแปลงกาย

ความสามารถของออบเจกต์ต่างคลาสที่สามารถตอบสนองต่อชื่อเมธอดเดียวกันในรูปแบบที่แตกต่างกัน ผ่านการ Overriding หรือ Interface

4. Abstraction

นามธรรม / ซ่อนความซับซ้อน

ซ่อนรายละเอียดการทำงานที่ซับซ้อนไว้เบื้องหลัง แสดงให้เห็นเฉพาะโครงสร้างหลักที่จำเป็นผ่าน abstract class หรือ interface

ตัวอย่างโครงสร้าง PHP OOP

สัมผัสความเรียบง่ายและเป็นระบบของการเขียนโปรแกรมเชิงวัตถุด้วย PHP 8+ คลาส Fruit แสดงถึงการประกาศคุณสมบัติ, เมธอด, Constructor และการสร้างออบเจกต์เพื่อใช้งานจริง

  • ✨ ใช้ __construct() เพื่อกำหนดค่าตั้งต้นให้ออบเจกต์ทันที
  • 🔒 ใช้ Access Modifier private ป้องกันการแก้ไขตัวแปรโดยตรง
  • 🚀 เรียกใช้เมธอดด้วยตัวดำเนินการ ->
ดูตัวอย่างเพิ่มเติมในบทที่ 2
Fruit.php 
<?php
// ประกาศคลาส (Class)
class Fruit {
    // Properties (คุณสมบัติ)
    private $name;
    private $color;

    // Constructor (เมธอดทำงานอัตโนมัติตอนสร้างออบเจกต์)
    public function __construct($name, $color) {
        $this->name = $name;
        $this->color = $color;
    }

    // Methods (พฤติกรรม)
    public function getInfo() {
        return "ผลไม้: " . $this->name . " (สี" . $this->color . ")";
    }
}

// การสร้าง Object จาก Class
$apple = new Fruit("แอปเปิ้ล", "แดง");
$banana = new Fruit("กล้วย", "เหลือง");

echo $apple->getInfo();  // ผลลัพธ์: ผลไม้: แอปเปิ้ล (สีแดง)
echo $banana->getInfo(); // ผลลัพธ์: ผลไม้: กล้วย (สีเหลือง)
?>
